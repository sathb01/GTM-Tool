# GTM OS Asset Rules

## Purpose

Assets turn a completed part of the intake into a reusable, shareable working document. They should help a user act, not repeat the full report.

## Availability

- An asset uses the currently saved record as its source of truth.
- The ICP Brief becomes available once a priority customer group has been named or selected.
- Pre-revenue records can use the ICP Brief as a hypothesis to validate. Post-revenue records can use it as an ICP candidate or validated ICP, depending on captured evidence.

## Content Rules

- Every asset must name its purpose, source inputs, and status.
- Facts captured in the intake must be distinguished from recommendations and evidence gaps.
- An asset must not invent firmographics, buyer roles, proof, or validation. Missing information is shown as an evidence gap or a next question.
- An asset must be concise enough to use in work, with grouped bullets instead of raw intake dumps.
- The ICP Brief must include: priority segment, qualification/search criteria, buyer problem and use case, urgency, buying roles, observable signals, disqualifiers, supporting evidence, open evidence gaps, and a first activation step.

## Export Rules

- Every asset view must provide a Download PDF action.
- The PDF export should contain the selected asset only, not the entire action plan.
- The export should use the browser print dialog so the user can save a standard PDF without needing a separate account or service.

## Refresh Rules

- Assets are generated from the saved intake when opened; they are not disconnected snapshots.
- Updating the intake refreshes the asset the next time it is opened or exported.
- Existing saved answers are preserved through the app migration layer when the intake structure changes.
